~ Comptabilité et Facturation Laurux ~


 

    Configuration accès base de données et tunnel ssh


    Cet écran permet de saisir les informations de connexions à la base de données.
    Il se divise en deux parties distinctes concernant les informations de connexion à la base de donnée et celles relatives a SSH.

    En premier lieu on saisira le nom de l'utilisateur MySQL et son mot de passe tel qu'ils ont été définis par la commande Grant durant l'installation.
    Viendra ensuite la saisie de l'adresse du serveur MySQL ainsi que son port.

    Si la connexion à MySQL doit se faire via un tunnel SSH alors il sera nécessaire de cocher le bouton "Connexion MySQL par SSH".
    Cette action ouvrira les champs de saisie suivants:
    Le nom de l'utilisateur SSH ainsi que son mot de passe.
    L'adresse du tunnel ainsi que son port d'accès sur le serveur et son port d'accès en connexion locale.

    Important: La commande de connexion à SSH utilise l'utilitaire "sshpass" pour transmettre le mot de passe à SSH. Il faut donc impérativement installer ce logiciel pour utiliser un tunnel SSH.

    Dans le cas d'une utilisation multi-utilisateurs, il faudra définir le poste sur lequel se feront les mises à jour de la base (généralement sur le serveur Mysql) et activer sur ce dernier le bouton "A cocher si c'est le cas". En monoposte ce bouton devra être toujours être coché.

    Si la base est stockée chez un hébergeur qui impose un préfixe dans les noms des bases il sera possible de modifier le nom de la base Laurux.
    Voici comment procéder :
    1- Faire une sauvegarde de votre base. Cela va générer un fichier nommé Laurux01.sql (01 représentant le numéro de la société de travail)
    2- Renommer ce fichier avec votre préfixe. Par exemple Lrx_Laurux01.sql. Attention ! Veiller a bien conserver le nom initail du fichier (dans notre exemple Laurux01)
    3- Lancer Laurux, aller dans les préférences et cocher le bouton "Gestion des préfixes" dans le premier onglet.
    4- Aller en base ==> connexion nouvelle base  puis modifier le nom de la base en y mettant le préfixe.
    5- Le programme va créer la base puis fermer Laurux
    6- Relancer Laurux, aller en Base ==> Restauration et restaurer la base (Dans notre exemple Lrx_Laurux01.sql)

    N'oubliez pas d'enregistrer à chaque modification.
